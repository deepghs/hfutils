hfutils.meta.version

This module provides functionality to retrieve information about Hugging Face sites and their versions.

It contains utilities to identify whether a site is the official Hugging Face hub, a Hugging Face enterprise deployment, or a custom self-hosted deployment that maintains API compatibility with Hugging Face. The module distinguishes between:

  • Official Hugging Face (huggingface.co): The main public hub

  • Hugging Face Enterprise: Custom deployments supported by Hugging Face but not the official site

  • Self-hosted compatible projects: Open source projects that implement HF-compatible APIs

The identification mechanism works by checking the /api/version endpoint. Official huggingface.co returns 401 for this endpoint, while enterprise and self-hosted deployments return site metadata.

HfSiteInfo

class hfutils.meta.version.HfSiteInfo(name: str, api: str, version: str, endpoint: str)[source]

Data class containing information about a Hugging Face site.

This class holds metadata about a Hugging Face deployment, including the site identifier and version information. It can represent:

  • Official Hugging Face hub (site=’huggingface’, version=’official’)

  • Hugging Face enterprise deployments (site=’huggingface’, version=’custom’)

  • Self-hosted compatible projects (site=custom_name, version=custom_version)

Parameters:

  • name (str) – The human-readable name of the site deployment

  • api (str) – The site identifier (e.g., ‘huggingface’ for official hub or enterprise)

  • version (str) – The version of the site deployment (‘official’, ‘custom’, or specific version)

  • endpoint (str) – The API endpoint URL of the site

Example:

>>> # Official Hugging Face hub
>>> site_info = HfSiteInfo(name='HuggingFace (Official)', api='huggingface', version='official', endpoint='https://huggingface.co')
>>> print(f"{site_info.api} ({site_info.version})")
huggingface (official)

>>> # Self-hosted project
>>> site_info = HfSiteInfo(name='Custom Hub (1.2.3)', api='custom-hub', version='1.2.3', endpoint='https://my-hub.com')
>>> print(f"{site_info.api} v{site_info.version}")
custom-hub v1.2.3

hf_site_info

hfutils.meta.version.hf_site_info(endpoint: str | None = None, hf_token: str | None = None) → HfSiteInfo[source]

Retrieve information about a Hugging Face site deployment.

This function queries the /api/version endpoint to determine the type of deployment:

  1. Official Hugging Face (huggingface.co): Returns 401 for /api/version, identified as ‘official’

  2. Hugging Face Enterprise: Returns 401 for /api/version but not on official domain, identified as ‘custom’

  3. Self-hosted compatible projects: Returns custom site name and version information

The identification mechanism relies on the fact that the official huggingface.co and enterprise deployments return 401 for the /api/version endpoint without proper authentication, while self-hosted projects that maintain API compatibility provide this endpoint with site metadata.

Parameters:

  • endpoint (Optional[str]) – The API endpoint URL. If None, uses the default Hugging Face endpoint

  • hf_token (Optional[str]) – The Hugging Face authentication token for private endpoints

Returns:

Site information including site identifier and version

Return type:

HfSiteInfo

Raises:

requests.HTTPError – If the API request fails (except for 401 on Hugging Face deployments)

Example:

>>> # Query official Hugging Face hub
>>> info = hf_site_info()
>>> print(f"{info.api} ({info.version})")
huggingface (official)

>>> # Query Hugging Face enterprise deployment
>>> info = hf_site_info(endpoint='https://company.huggingface.co')
>>> print(f"{info.api} ({info.version})")
huggingface (custom)

>>> # Query self-hosted compatible project
>>> info = hf_site_info(endpoint='https://my-custom-hub.com')
>>> print(f"{info.api} v{info.version}")
my-custom-hub v2.1.0
